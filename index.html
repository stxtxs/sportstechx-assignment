01
SQL · PostgreSQL
Fix a Slow Query
⏱ ~15 min
This query runs on a startups table with ~150,000 rows. It works but times out under load. Spot what's wrong and fix it.
SQL 
SELECT id, name, country, category
FROM startups
WHERE name ILIKE '%sports%'
  AND category = 'analytics'
  AND country = 'India'
ORDER BY created_at DESC
LIMIT 20;
Your Tasks
  • Identify what's making this slow — a short explanation (3–5 sentences) is enough.
  • Write the SQL to fix it. Use any PostgreSQL feature you think is appropriate — indexes, full-text search, whatever makes sense.
Deliverables
  • A q1.sql file with your fix and any DDL needed.
  • A few lines in your README explaining your reasoning.

04
Code Quality · Refactoring
Clean Up This Component
⏱ ~15 min
The component below works, but it was written in a rush. Refactor it the way you would in a real PR review — we're not looking for a complete rewrite, just the improvements that matter.
TSX 
export function StartupCard({ data }: { data: any }) {
  const [show, setShow] = useState(false);
  const [d, setD] = useState(null);

  useEffect(() => {
    fetch('/api/startups/' + data.id)
      .then(r => r.json())
      .then(res => setD(res));
  }, []);

  const money = (n: any) => {
    if (n >= 1000000) return '$' + n / 1000000 + 'M';
    if (n >= 1000) return '$' + n / 1000 + 'K';
    return '$' + n;
  };

  return (
    <div>
      <h2>{data.name}</h2>
      <p>{data.country} · {data.category}</p>
      { d && <p>Raised: {money(d.total_raised)}</p> }
      <button onClick={() => setShow(!show)}>
        {show ? 'Hide' : 'Details'}
      </button>
      { show && <p>{data.description}</p> }
    </div>
  );
}
Your Tasks
  • Refactor the component — fix the typing, naming, side effects, and anything else you'd flag in a code review. Don't change what it does, just how it's written.
  • Add a comment block at the top listing the issues you found and what you changed.
Deliverables
  • A single refactored StartupCard.tsx with your issue notes at the top.
